## POS Shop – Multi‑tenant Next.js POS

**POS Shop** is a multi‑tenant POS system built with **Next.js App Router**, **TypeScript**, **Tailwind** and **Prisma**.  
It is designed so you can host it once and onboard multiple businesses (cafes, mobile shops, clothing stores, etc).

### Features

- **Multi‑tenant core**: `Tenant`, `Location`, `User` with roles, `Product`, `Customer`, `Sale`, `SaleItem`, `Payment` in Prisma.
- **Authentication**: Email + password with secure hash, cookie‑based sessions, auth middleware, per‑tenant data isolation.
- **Owner onboarding**: `/register` flow creates a new tenant, owner user and main location.
- **POS dashboard**: `/dashboard` with revenue tiles and recent sales for the current tenant.
- **Products**: `/products` for listing and creating products scoped to a business.
- **Customers**: `/customers` to maintain basic customer profiles.
- **POS screen**: `/pos` to create simple sales and record them against the database.

---

## 1. Prerequisites

- **Node.js** 18+
- **npm** 10+
- A **PostgreSQL** database (local or hosted)

---

## 2. Install dependencies

```bash
cd pos-shop
npm install
```

---

## 3. Configure environment

Create a `.env` file in the project root (already generated by Prisma) and set:

```bash
DATABASE_URL="postgresql://USER:PASSWORD@HOST:PORT/DB_NAME"
SESSION_SECRET="a-long-random-secret-string"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

- `DATABASE_URL` should point to your PostgreSQL instance.
- `SESSION_SECRET` is used to sign session cookies.
- `NEXT_PUBLIC_APP_URL` should match how you access the app in the browser.

> The Prisma CLI initially put a demo `DATABASE_URL` in `.env` for a temporary Postgres. Replace it with your own.

---

## 4. Prisma: migrate & generate

Apply the schema and generate the Prisma client:

```bash
npx prisma migrate dev --name init_pos_schema
npx prisma generate
```

This will create the database tables for tenants, users, products, inventory, sales, customers, etc.

---

## 5. Run the app

```bash
npm run dev
```

Open `http://localhost:3000` in your browser.

---

## 6. Using the POS

1. **Register a business**
   - Go to `/register`.
   - Enter business name, slug, owner details and password.
   - This creates a `Tenant`, owner `User` and a default `Location`.

2. **Sign in**
   - Go to `/login` and sign in with the email and password you registered.

3. **Dashboard**
   - After login you land on `/dashboard` with revenue tiles and recent sales.

4. **Add products**
   - Navigate to `/products`.
   - Add products (name, SKU, price, optional stock tracking).
   - All products are scoped to your tenant.

5. **Add customers (optional)**
   - Navigate to `/customers` to create customer profiles.

6. **Make a sale**
   - Go to `/pos`.
   - Pick an optional customer.
   - For now, the checkout form accepts simple `productId:qty` pairs to create a sale.
     - (In a production version this becomes a full cart UI with clickable products and barcode support.)
   - Submitting the form creates a `Sale` with `SaleItem` rows linked to your tenant and user.

---

## 7. Multi‑tenant behaviour

- Every user belongs to exactly one `Tenant`.
- All key queries (`Product`, `Customer`, `Sale`, etc.) are filtered by `tenantId`.
- The middleware protects all app routes except auth and redirects unauthenticated users to `/login`.
